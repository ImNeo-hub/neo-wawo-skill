# 蒸馏手册 /wawo-learn：把原始经历库炼成「经历卡片」

> **这是可选工具，不是 /wawo 的前置条件。** /wawo 现在靠纯对话挖掘，不依赖预存卡片。但如果你手上有原始经历库（比如旧的 AI 对话记录），/wawo-learn 可以把里面你自己说的话提取成卡片，给未来的采访多一层底子。

原始经历库往往是 AI 对话全文堆（用户的话 + AI 的输出混在一起，AI 输出占大头）——直接在里面挖选题，挖出来的多是「知识选题」（从 AI 讲课内容长出来的，库里没有用户对应的真实故事，写起来是空心的）。

**解法：** `/wawo-learn` 离线把原始库蒸馏成「经历卡片」，供 /wawo 在有积累时参考。

## 蒸馏铁律
1. **只提取用户说的话，和从用户的提问里能稳定推断的处境。** AI 的回答、建议、方案、讲课内容**一律丢弃**——AI 输出唯一的价值是佐证用户当时在折腾什么。
2. **每条信息标注来源**：`用户原话`（直接引用，保留原文）或 `推断`（从提问/上下文推出的处境，措辞要克制）。
3. **绝不评判、绝不说教。** 债务、健康、心理、瘾这类内容是素材，原样记录，不加道德修饰。
4. 原始库**只读**；卡片库（私有目录）**可写**——这是两个库权限的关键区别。

## 卡片格式（统一经历卡片）

不分厚薄，统一编号 `Exxx`，全库连续编号。每条记录包含：

```
## E001｜日期｜一句话点出这件事
标签：#主题1 #主题2
深度：★☆☆ / ★★☆ / ★★★

原话：「……」「……」
推断：……

可追的线头：
- 问题1
- 问题2
```

**深度标准：**
- ★☆☆ 提了一嘴（只有一句话或一个事实，没有场景和细节）
- ★★☆ 聊过有细节（有具体场景、情绪、转折，但还能再深）
- ★★★ 已深采可写（有画面感的场景 + 真实反应 + 足够的细节，可以直接出初稿）

**标签规范：** 用中文短词，如 `#成长` `#学业` `#职业` `#创业` `#感情` `#金钱` `#自我认知` `#心理` `#自控` `#旅行`。一条记录可挂多个标签，交叉处往往是好选题。

**「可追的线头」：** 不是「回采问题」——不是说这条卡「不合格需要补」，而是「下次聊到这个方向时可以顺着追的问题」。★★★ 的卡也可以有线头（故事写完了还能往别的方向延伸）。

## 卡片库文件（私有目录，软链进 `memory/cards/`）
- `经历档案.md` —— 所有经历卡片，统一存放
- `README.md` —— 本次蒸馏的规则和日期（如有）

## 增量更新
- 新经历直接追加到 `经历档案.md`，编号续接。
- 同一件事的新细节**并入旧卡**（注明补充日期），同时更新深度星级。
- 对照已有卡片防止重复建卡。

## 深度升级（自然发生，不需要专门流程）
采访中聊到了某条 ★☆☆ 的经历，补了新细节 → 直接在原卡上加内容、把深度改成 ★★☆ 或 ★★★。不需要「升卡」这个动作——深度随对话自然增长。

## 卡片库的长期生长（比存量蒸馏更重要）
存量对话是一次性的矿，挖完边际价值递减。长期卡片主要靠**用户日常随口讲的新鲜事**：用户说「这周这事烦死我了」→ 当场建一张新卡（日期 + 事 + 原话 + 标签）。新鲜的一手叙述天然深度高，质量远高于从旧对话里反推。

## 隐私
卡片库含敏感内容（债务、健康、心理等），永远在仓库外私有目录，软链 + `.gitignore` 屏蔽，绝不上传。
