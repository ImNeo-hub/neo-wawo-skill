# neo-wawo-skill 验收标准

> Loop QA 验收文件。Claude 在 `/loop-qa neo-wawo-skill` 时读这个文件，逐项检查。

## 基本信息
- **验收对象**: 采访过程质量 + 初稿输出质量
- **自动化程度**: 低（采访是对话，难以自动化；检查主要靠规则审计）
- **前置**: `memory/wawo-state.md` 存在

---

## 测试场景

### S1: 冷启动（Phase 1 深度了解）

**输入**: 首次运行 `/wawo`，wawo-state.md 为空或不存在
**操作**: 进入 Phase 1 批量问答
**期望输出**: 每轮 30 个问题，覆盖主题路线图

**检查清单**:
- [ ] `AUTO` wawo-state.md 被创建/更新，phase = `1-in-progress`
- [ ] `AUTO` 第一轮问题数量 = 30
- [ ] `AUTO` 开场白包含「记者」「30 个问题」「10 轮」关键词
- [ ] `MANUAL` 问题覆盖当轮主题方向（对照 SKILL.md 的主题路线图）
- [ ] `MANUAL` 问题具体、不空泛（「你最近在读什么」✓ vs「你对知识怎么看」✗）
- [ ] `MANUAL` 无说教/评判语气

**通过条件**: 自动检查全过 + 问题质量人工审核通过

---

### S2: Phase 2-8 采访挖掘

**输入**: wawo-state.md phase = `mining`，用户说「最近在折腾 XX」
**操作**: 进入采访模式
**期望行为**: 一次只问一个问题，顺着上一句话的线头追

**检查清单**:
- [ ] `RULE` **一次只问一个问题**（铁律！Phase 2-8 绝不批量提问）
- [ ] `RULE` 只追问不评价（无「真的很厉害」「你可以这样想」）
- [ ] `RULE` 问题从用户上一句话中长出来（不是预设题库）
- [ ] `MANUAL` 用了四类追问中的至少 2 种（细节钻/时序追/反差问/身体感受）
- [ ] `MANUAL` 找到了「有故事感的线头」（具体事/语气变化/跳过的 B）
- [ ] `MANUAL` 无说教、无人生建议

**通过条件**: 两条铁律不违反 + 追问质量通过

---

### S3: 初稿写作

**输入**: 采访完成（≥ 8 轮追问），素材充分
**操作**: 进入写稿模式（读 coach.md）
**期望输出**: 第一人称初稿

**检查清单**:
- [ ] `RULE` **每个细节都来自采访原文**（绝不编造）
- [ ] `MANUAL` 第一人称视角
- [ ] `MANUAL` 有具体场景（不是抽象感悟堆叠）
- [ ] `MANUAL` 留白标注了采访中没采到的地方（「这里需要你补一句」）
- [ ] `MANUAL` 故事有转折或冲突（不是流水账）
- [ ] `MANUAL` 读起来像真人说的（有毛边、有犹豫）

**通过条件**: 绝不编造铁律不违反 + 初稿有血有肉

---

### S4: 状态一致性

**输入**: 任何阶段中途退出后重新进入
**操作**: 重新调用 `/wawo`
**检查清单**:
- [ ] `AUTO` 读取 wawo-state.md 并判断正确路径
- [ ] `AUTO` Phase 1 续接时，读 rounds_completed 从断点继续（不重新问已问过的）
- [ ] `AUTO` Phase 2-8 续接时，读上次话题继续（不从零破冰）
- [ ] `MANUAL` 逃生舱生效：用户带明确话题来 → 直接进 Phase 3/5

## 规则审计脚本（Claude 内部执行）

Claude 在验收采访输出时，对自己的回复做以下检查：

```
规则审计清单（逐条扫描 Claude 的每一条回复）:

1. 问题计数: 这条回复里有几个问号？
   - Phase 1: 应该 = 30（批量模式）
   - Phase 2-8: 应该 ≤ 1（一次一问铁律）
   - > 1 且在 Phase 2-8 = ❌ 违规

2. 评价检测: 这条回复里有没有以下句式？
   - 「你做得很好/很厉害/很棒」 = ❌
   - 「你可以/应该/不妨试试」 = ❌ (说教)
   - 「其实你是/你本质上是」 = ❌ (评判)
   - 「我觉得你」 = ⚠️ (可能在评判)

3. 编造检测（初稿阶段）: 初稿中的每个具体细节，
   能否在采访记录中找到对应原文？
   - 找不到对应 = ❌ 编造
   - 找到但加了修饰 = ⚠️ 需确认

4. 隐私检测: 回复中有没有在非采访场景引用
   用户在 wawo 中的敏感回答？
   - 有 = ❌ 违规
```

## 常见失败 & 修复

| 失败 | 原因 | 修复 |
|------|------|------|
| Phase 2-8 批量提问 | 没区分 Phase 1 和 Phase 2-8 的问法 | SKILL.md 已明确：Phase 1 批量，Phase 2-8 一次一问 |
| 评价/说教 | 默认想「帮」用户 | 只追问不评价——他的纠结是素材不是问题 |
| 初稿编造细节 | 采访素材不够就补了 | 宁可留白标注「这里需要你补一句」 |
| 断点丢失 | wawo-state.md 没更新 | 每轮结束后立即写回 state |
| 问题太大 | 问了「你觉得人生意义是什么」 | 回到小的、具体的、今天能答的 |

## 模拟测试（模式 D）

> 画像文件位于 `~/Documents/neo-test-personas/`。
> 推荐画像：`persona-buyu.md`（测真实场景）、`persona-xiaobai.md`（测冷启动）。

### SIM-1: 冷启动第一轮（persona-xiaobai）
**目的**: 验证 skill 对完全陌生的用户能否正常工作
**画像**: xiaobai（回答偏短、需要追问才展开）
**场景**: Phase 1 第一轮 30 题
**重点观察**:
- 开场白是否亲切不吓人
- 30 个问题是否覆盖当轮主题
- 面对短回答是否有耐心

### SIM-2: Phase 2-8 采访深挖（persona-buyu）
**目的**: 验证采访引擎的追问质量
**画像**: buyu（触发点丰富、会自嘲回避、会反问测试边界）
**场景**: 从「最近在折腾 AI 做内容」开始，跑 8 轮追问
**重点观察**:
- 一次只问一个问题（铁律）
- 碰到「算了不说了」时是否追过来
- 碰到「你觉得呢」时是否不越界
- 是否在连环自问中找到真正的线头
- 是否用到了至少 2 种追问类型

### SIM-3: 初稿写作（persona-buyu）
**目的**: 验证从采访素材到初稿的转化质量
**画像**: buyu
**场景**: 基于 SIM-2 的 8 轮采访素材，写初稿
**重点观察**:
- 每个细节是否来自采访原文（绝不编造）
- 是否留白标注了未采到的地方
- 是否有真实毛边（不完美的真人感）

## Loop 使用方式

```bash
# 单次验收（审计最近一次 wawo 会话）
/loop-qa neo-wawo-skill

# 模拟测试（用画像跑对话模拟）
/loop-qa neo-wawo-skill    # 选模式 D

# Loop 迭代改进
/loop 用 persona-buyu 模拟 wawo Phase 2-8 采访 8 轮，跑规则审计，有问题就改 interview.md，改完重新模拟

# 改进采访引擎
/loop 读 interview.md 的四类追问，用 persona-xiaobai 模拟一个短回答用户，检查 skill 能否把她打开
```
